# SHOWRUNNER 로드맵

MVP 이후 발전 방향을 단계별로 정리합니다. 각 단계는 독립적으로 배포 가능하도록 설계했습니다.

## Phase 1 — 운영 안정화 (MVP 직후)

- **Prisma + PostgreSQL 전환**: `prisma/schema.prisma`는 이미 정식 작성되어 있음. `src/lib/db/store.ts`의 함수 시그니처를 유지한 채 내부를 `@prisma/client` 호출로 교체
- **정식 계정 시스템**: 현재는 이름+4자리 PIN 수준의 프로필 잠금(세션 쿠키 서명, 프로젝트/레슨런 프로필 단위 격리)까지만 구현됨. 이메일 인증, 비밀번호 재설정, 로그인 시도 제한(rate limiting)을 추가
- **워크스페이스 내 역할 구분**: 관리자/멤버 등 역할별 권한, 프로필(팀) 내 여러 사람 초대
- **레슨런 조직 공용 라이브러리**: 지금은 프로필 단위로만 과거 사례가 공유됨. "공용 라이브러리에 등록" 같은 명시적 opt-in으로 팀 간 선택적 공유 지원
- **파일 저장소 교체**: `FileStorage` 추상화를 유지한 채 S3/GCS 어댑터 추가
- **PDF 리포트 출력**: 현재 Markdown 내보내기를 PDF 렌더링(서버사이드)으로 확장
- **감사 로그**: 문서 반영/위험 수정/시뮬레이션 결정 이력 추적

## Phase 2 — AI 고도화

- **임베딩 기반 RAG**: 로컬 term-overlap 저장소를 pgvector 또는 전용 벡터 DB로 교체 (`getRelevantChunks()` 시그니처 유지)
- **직접 입력 대응안 LLM 평가**: 사용자가 자유 입력한 대응을 Claude가 상황 맥락·매뉴얼 근거와 대조해 효과를 산정
- **문서 충돌 해소 보조**: 충돌 항목에 대해 "어느 문서가 더 최신/신뢰할 만한지" 근거를 제시 (자동 병합은 계속 금지)
- **시나리오 생성 다양화**: 템플릿 16종 → 행사 유형별 템플릿 팩 + LLM 동적 생성 혼합
- **과거 사례 학습 루프**: 시뮬레이션 리포트에서 검증된 대응을 HistoricalCase로 역저장 (사용자 승인 필수)

## Phase 3 — 시뮬레이션 심화

- **연쇄 깊이 확장**: 현재 깊이 1 제한을 설정 가능한 깊이(최대 3)로 확장, 연쇄 트리 시각화
- **멀티롤 시뮬레이션**: 무대감독/음향/클라이언트 응대 등 역할별 분기 판단
- **실시간 모드**: 상황 트리거 시각 점프 방식 → 배속 조절 가능한 연속 시간 진행
- **팀 시뮬레이션**: 여러 사용자가 각자 역할로 동시 참여, 판단 충돌 기록
- **난도 자동 조정**: 과거 시뮬레이션 점수 기반 적응형 난도

## Phase 4 — 조직 지식화

- **행사 지식 베이스**: 프로젝트 간 사례/체크리스트 공유 라이브러리
- **매뉴얼 자동 개정 제안**: 리포트의 "매뉴얼 반영 제안"을 원문 문서의 수정 diff로 생성
- **대시보드**: 에이전시 단위 준비도 추이, 반복 위험 카테고리 통계
- **외부 연동**: 캘린더(행사 일정), Slack(상황 알림 훈련), Notion(체크리스트 내보내기)

## 기술 부채 상환 계획

| 항목 | 현재 상태 | 목표 |
|---|---|---|
| 데이터 레이어 | JSON 파일 저장소 | Prisma + PostgreSQL |
| 벡터 검색 | term-overlap | pgvector |
| 백그라운드 처리 | fire-and-forget Promise | 큐(BullMQ 등) + 재시도 |
| 테스트 | 엔진 단위 12건 | API 통합 + E2E(Playwright) |
| 국제화 | 한국어 고정 | i18n 레이어 분리 |
